# Responsive Holy Grail Layout with Grid

For this exercise, we are going to take our Holy Grail Layout from the first exercise and make it responsive. To do this we will simply change our fixed track sizes to be dynamic. Consider this a warmup for the next exercise and make sure your layout is responding properly when resizing the browser window.

### Hints
- You only need to add to the CSS selectors
- Use dynamic track sizes for your columns and rows

When the browser is narrow:

When the browser is stretched wide:

![desired outcome wide](./desired-outcome-wide.png)

### Self Check
- The gap is 15px
- The grid has two columns
- The grid has four rows
- The grid tracks do not use static sizes (no pixels!)
- The second column is three times larger than the other
- The third row is five times larger than the others
- The columns narrow and widen as the viewport changes size
- Row 3 gets taller to accomodate text wrapping as you narrow the viewport size

# Layout Holy Grail Responsivo con Grid

Per questo esercizio, prenderemo il nostro Layout Holy Grail dal primo esercizio e lo renderemo responsivo. Per farlo, cambieremo semplicemente le nostre dimensioni fisse delle tracce in dimensioni dinamiche. Considera questo come un riscaldamento per il prossimo esercizio e assicurati che il tuo layout risponda correttamente quando ridimensioni la finestra del browser.

### Suggerimenti
- Devi solo aggiungere ai selettori CSS
- Usa dimensioni dinamiche delle tracce per le tue colonne e righe

## Risultato Desiderato

Quando il browser è stretto:

![desired outcome narrow](./desired-outcome-narrow.png)

Quando il browser è allargato:

![desired outcome wide](./desired-outcome-wide.png)

### Autoverifica
- Lo spazio tra gli elementi (gap) è 15px
- La griglia ha due colonne
- La griglia ha quattro righe
- Le tracce della griglia non usano dimensioni statiche (niente pixel!)
- La seconda colonna è tre volte più grande dell'altra
- La terza riga è cinque volte più grande delle altre
- Le colonne si restringono e si allargano quando le dimensioni del viewport cambiano
- La riga 3 diventa più alta per accommodare il testo che va a capo quando restringi le dimensioni del viewport
